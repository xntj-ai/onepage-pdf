HTML → PDF · One continuous page

Turn an HTML page
into one continuous-page PDF

onepage-pdf is a Claude Code / Agent skill. Hand it a long scrolling HTML page and it produces a single tall PDF page — no pagination breaks, desktop layout and backgrounds preserved, text still selectable. Optional string redaction with a leak check.

A normal print-to-PDF chops a long page into A4 slices and cuts content at the page breaks. onepage-pdf takes the opposite stance: it never predicts the printed height. It renders onto an oversized page, measures the real content bottom from the rendered PDF, and crops to fit — so the height is exact by construction. Just Python and a local Chrome; no Playwright, no service.

Paginated vs. one continuous page

The same long page, exported two ways — that is the entire point of the skill.

Normal print-to-PDF

Sliced into A4 pages

  • Content severed mid-line at every A4 break
  • Breakpoints fire at ~741px and collapse the grid
  • Phantom pages from px→pt rounding
onepage-pdf

One tall, continuous sheet

  • No breaks — one sheet from top to bottom
  • Desktop layout & backgrounds preserved
  • Cropped to the exact content height

Core features

One continuous page, never paginated

Renders onto an oversized bedrock page, then crops MediaBox + CropBox to the real content height. No A4 cuts, nothing sliced mid-line.

Height cropped, not calculated

The printed height is measured from the rendered PDF, never guessed from screen scrollHeight — exact by construction.

Layout & backgrounds preserved

print-color-adjust: exact keeps gradients; fixed backgrounds are normalized; scroll-reveal animations are forced visible.

Redaction with leak check

Substitute strings before rendering and assert a forbidden word list never survives in the HTML or the final PDF text.

When to use

Good fit

One clean PDF to send

  • HTML proposals, reports, landing pages that must leave the browser as one PDF
  • Deliverables sent in a chat app or email that have to open cleanly on a phone
  • AI-generated PPVI-style pages that look right on desktop
Not for

Genuine print / pixel capture

  • A print job that actually wants paginated A4 — just print normally
  • Pixel-faithful WebGL / canvas / perspective scenes the print path rasterizes away
  • Pages that defeat both crop heuristics (switch to the CDP route)

Quick start

Install into your Claude Code skills directory

git clone https://github.com/xntj-ai/onepage-pdf.git ~/.claude/skills/onepage-pdf
pip install pymupdf

Ask Claude, or run the script

Say "use onepage-pdf to turn this proposal into a single continuous-page PDF" — Claude inspects the HTML, writes the breakpoint locks, and converts. Or call it directly: python scripts/onepage_pdf.py input.html -o output.pdf --width 1280.

Eyeball the result

The script prints OK 1 page, WxHpt. Render a thumbnail to confirm the layout is intact, backgrounds are present, and nothing is cut at the bottom.
